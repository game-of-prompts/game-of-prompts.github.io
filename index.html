Technological Foundations
Celaut Architecture
A system for decentralized execution of computational services that enables immutable and uniquely identified services, isolated execution in containers, and deterministic reproducibility—critical for fair competition in GoP.
Ergo Blockchain
A Proof-of-Work blockchain with extended UTXO model and Sigma Protocol-based smart contracts, securing gameplay, transparently recording results, and enabling fair prize distribution.
These technologies combine to create a trustless environment where games and solutions can be verified and evaluated without centralized authority, maintaining the platform's P2P philosophy.
Why use GoP?
Game of Prompts establishes a clear incentive framework for both creators and players, fostering a dynamic and rewarding competitive environment.
For game developers
Earn commissions from prize pools and gain valuable insights from diverse AI solver-services designed to tackle your games. This provides a rich dataset for optimizing your creations.
For players
Pay a small participation fee to enter the competition. The player with the highest-scoring solution will claim the entire prize pool, rewarding skill and innovation.
Core Components
Game Service
Created by game devs.
An autonomous service that encapsulates a game's logic and the secret. It evaluates solver performance, generates scores, and creates the cryptographic commitments needed for blockchain validation.
Solver Service
Created by players.
Implements strategies to maximize scores in specific games. The solver is packaged and sent to the Game Service, which executes it in a secure, isolated environment for evaluation.
GoP Web
A community portal where users discover games, read rules, download Game Services, and publish their results on the Ergo blockchain.
It supports self-hosting to enable a fully trustless, peer-to-peer experience.
Game Service and Solver Service
The Game Service and Solver Service must be run on a local Celaut node, which can operate entirely offline without any network connection.
GoP Web
The GoP Web portal can be accessed via a GitHub.io instance or run locally as a Celaut service.

Security
Transparent Yet Private Competition
Game of Prompts uses advanced cryptography to ensure fair competition while protecting participants' strategies:
Code Protection
The game's intellectual property and solution are protected through obfuscation to prevent reverse engineering.
Private Results
Results are concealed using cryptographic commitments, ensuring a private and unbiased competition.
Immutable Proof
Final validation on the Ergo blockchain generates a public, immutable, and auditable proof of victory.
Score Validation Mechanism
1
Player Participation
Player publishes their participation on the Ergo blockchain.
2
Creator Reveals Secret
After deadline, creator reveals the game secret in the resolution transaction.
3
Smart Contract Validation
The game contract computes a commitment for each score in the score list using the solver ID, the score value, hashed logs, and the revealed secret.
4
Score Verification
When the score commitment matches the participation commitment, that score is validated as authentic for that participation. The highest validated score wins.
5
Winner is revealed
The highest validated score wins.
This deterministic on-chain process ensures transparency while preventing manipulation by either players or creators.
Optional game features

Poker mode

Strategic participation with risk and reward How it works: Participants choose how much to pay (above a set minimum). Higher payment = higher final score (e.g. ×2, ×5, etc.). Players can submit multiple decoy scores. Poker-style strategy: Some may bluff: pay a high fee + submit fake high scores to intimidate others. Introduces mind games and strategic pressure. Game creator control: Decides whether to enable Poker Mode. Sets the score multiplier based on fee paid.

Resource Limitation

This feature adds a technical challenge by constraining the resources a player's solution can use. How it Works: The game creator can set specific limits on computational resources. Types of Limits: Constraints can include maximum RAM usage, CPU time, or the number of service dependencies allowed. Goal: This forces players to develop not just a working solution, but a highly efficient and optimized one, adding a significant engineering challenge to the game.

What Can be a Game?
Types of Games You Can Create or Compete In:
Classic Arcade Challenges
(e.g., Pac-man, Snake like)
The AI solver will control the character in fast-paced, skill-based game environments.
Score: Game points, survival time, or levels cleared.
Open World Discovery
(e.g., Minecraft-like)
Explore, build, and adapt in rich voxel-based worlds. Recycle the same world into many different challenges.
Score: Resource efficiency, map exploration, or mission complexity.
Financial Trading Simulations
Bot vs. market. Trade virtual assets in realistic simulations using historical or synthetic data.
Score: Net profit, Sharpe ratio, drawdown, or performance vs. market benchmarks.
Protein Folding Science Games
Predict 3D protein structures from amino acid sequences. A real scientific challenge for AI solvers.
Score: Stability (energy), structural accuracy, and folding efficiency.
Strategy & Resource Management
Build, optimize, and dominate using limited resources. Think StarCraft meets spreadsheets.
Score: Speed, efficiency, economic output, or resilience.
Code Golfing & Optimization
Write the fastest, shortest, or most elegant code for classic algorithmic challenges.
Score: Runtime, byte length, or operation count.
Natural Language Understanding
Parse, interpret, and act on instructions in plain text. A Turing test with rules.
Score: Task completion accuracy, response correctness, or extraction relevance.
Data Science & Database detective
Wrangle and query massive, complex databases to solve data-driven mysteries.
Score: Query performance, result accuracy, or insight relevance.
